Quickstart

Orbicheck is API-first. Get an API key from your dashboard and hit our endpoints from your backend or edge.

# 1) Set your API key
export ORBICHECK_KEY=oc_live_xxx

# 2) Validate a VAT ID
curl -s https://api.orbicheck.com/v1/validate/vat \
  -H "Authorization: Bearer $ORBICHECK_KEY" \
  -H "Content-Type: application/json" \
  -d '{ "vat": "ESB12345678" }' | jq .

# 3) Standardize a phone number (E.164)
curl -s https://api.orbicheck.com/v1/validate/phone \
  -H "Authorization: Bearer $ORBICHECK_KEY" \
  -H "Content-Type: application/json" \
  -d '{ "phone": "600 123 456", "country": "ES" }' | jq .

# 4) Dedupe customer
curl -s https://api.orbicheck.com/v1/dedupe/customer \
  -H "Authorization: Bearer $ORBICHECK_KEY" \
  -H "Content-Type: application/json" \
  -d '{ "email": "a@b.com", "name": "A B", "phone": "+34600123456" }' | jq .

SDKs are optional; simple JSON in/out. Use connectors for Shopify/Woo/BigCommerce/Magento if needed.

Authentication

Bearer tokens via Authorization header. Separate keys per project recommended.

Authorization: Bearer oc_live_xxx

Validation API

VAT

POST /v1/validate/vat
{
  "vat": "ESB12345678",
  "cache_max_age": 86400
}
# → { "valid": true, "country": "ES", "source": "VIES|cache|fallback", "reason_codes": [] }

Email/Phone/Address

POST /v1/validate/phone
{ "phone": "(600) 123-456", "country": "ES" }
# → { "valid": true, "e164": "+34600123456", "reason_codes": [] }

POST /v1/validate/address
{ "line1": "Calle Mayor 1", "city": "Madrid", "zip": "28013", "country": "ES" }
# → { "valid": true, "normalized": {...}, "reason_codes": [] }

ES-LATAM Tax IDs

POST /v1/validate/id
{ "id": "123.456.789-09", "country": "BR", "type": "CPF" }
# → { "valid": true, "normalized": "12345678909", "reason_codes": [] }

Dedupe API

POST /v1/dedupe/customer
{
  "record": {"email":"a@b.com","name":"Ana B","phone":"+34600123456"},
  "strategy": {"deterministic":["email","phone"], "fuzzy":["name"], "threshold":0.82}
}
# → { "matches":[{"id":"cus_123","confidence":0.91}], "suggest_merge": true }

Rules & Holds

POST /v1/rules/evaluate
{
  "order": {"id":"ord_123","cod":true,"shipping":{"address":{...}}},
  "customer":{"id":"cus_123","history":{"rto_rate":0.2}}
}
# → { "hold": true, "tags": ["COD_RISK"], "reason_codes": ["RTO_HEURISTIC_MATCH"] }

Webhooks

POST https://yourapp.com/webhooks/orbicheck
{ "type":"validation.completed", "data":{...}, "project":"proj_123" }

Errors & Reason Codes

Every response can include reason_codes suitable for exposure at checkout (human-friendly mappings available).

# Example
{ "valid": false, "reason_codes": ["VAT_NOT_FOUND","ADDRESS_CITY_MISMATCH"] }
